# 메이플스토리 + 계산기 (mCal) 시스템 아키텍처

"메이플스토리 + 계산기" (mCal) 시스템은 주로 **HTML, CSS, jQuery를 사용하는 JavaScript로 구축된 클라이언트 측 단일 페이지 애플리케이션(SPA)**으로 구현되었습니다. 이 시스템은 사용자의 웹 브라우저 내에서 전적으로 작동하며, 정적 파일을 제공하는 것 외에 계산 또는 동적 콘텐츠 생성을 담당하는 서버 측 구성 요소는 없습니다.

## 주요 구성 요소 및 역할:

### 1. `index.html` (프레젠테이션 및 구조)
*   **메인 진입점:** 전체 페이지 구조를 정의하는 루트 HTML 파일입니다.
*   **계산기 모듈:** 각 계산기 모듈(예: "메소와 메포 구매", "MVP작 비용 계산기", "보상분배계산기", "아이템의 현금가치", "과거와 현재 아이템 가치", "메소판매차익", "골드주화가치", "주흔&아즈모스코인가치", "피방조각효율")에 대한 개별 `<section>` 요소를 포함합니다.
*   **입력 및 표시:** 사용자 입력 필드(`input`) 및 동적 결과 표시 영역(`span`)을 포함합니다.
*   **리소스 로딩:** 애플리케이션에 필요한 모든 외부 및 로컬 CSS 및 JavaScript 리소스를 연결합니다.
*   **메타데이터:** 설명, 네이버 사이트 확인, 소셜 미디어 공유를 위한 오픈 그래프 프로토콜 등 SEO(검색 엔진 최적화)를 위한 `<meta>` 태그를 포함합니다.

### 2. `assets/css/` (스타일링)
*   **`main.css` 및 `index.css`**: 전반적인 시각적 모양, 레이아웃 및 계산기 요소에 대한 특정 스타일을 정의하는 사용자 지정 스타일시트입니다.
*   **`fontawesome-all.min.css`**: 사용자 인터페이스 전반에 사용되는 아이콘 라이브러리를 제공합니다.
*   **외부 CSS**: 향상된 테이블/그리드 스타일링에 사용되는 `tui-grid.css`와 같은 외부 스타일시트를 통합합니다(제공된 `index.html` 마크업에서 `tui.Grid` JavaScript 구성 요소 자체는 최소한으로 사용되거나 향후 확장을 위한 것으로 보입니다).
*   **Sass 전처리**: `assets/sass/` 내에 `_*.scss` 파일이 있다는 것은 Sass(Syntactically Awesome Style Sheets)가 CSS 전처리에 사용되어 최종 `.css` 파일로 컴파일될 가능성이 높다는 것을 나타냅니다.

### 3. `assets/js/` (클라이언트 측 로직 및 상호 작용)
*   **`index.js` (핵심 애플리케이션 로직):**
    *   **계산기 엔진:** 모든 메이플스토리 관련 계산기를 위한 전체 계산 로직을 포함합니다. 여기에는 게임 메커니즘에 특화된 복잡한 공식과 조건부 로직이 포함됩니다.
    *   **실시간 업데이트:** 다양한 양식 필드의 사용자 입력을 처리하고 페이지를 새로 고칠 필요 없이 실시간으로 계산 결과를 동적으로 업데이트하여 즉각적인 피드백을 제공합니다.
    *   **데이터 지속성:** 브라우저의 `localStorage`를 활용하여 사용자 입력 값을 저장하고 로드합니다. 이를 통해 브라우저 세션 간에 입력이 유지되어 사용자 편의성을 높입니다.
    *   **입력/출력 형식 지정:** 사용자 친화적인 숫자 입력(예: 천 단위 구분 기호에 쉼표 추가)을 위한 `onlyNumberWithComma` 및 대규모 숫자에 대한 특별 처리(예: 1억에 대해 "억" 추가)를 포함하여 로케일에 적합한 형식으로 결과를 표시하기 위한 `customFormatNumber`와 같은 유틸리티 함수를 포함합니다.
    *   **이벤트 처리:** 사용자 상호 작용(예: 입력 필드에 입력, 라디오 버튼 변경)을 기반으로 특정 계산기 함수의 실행을 조정합니다.
*   **`main.js` (UI/UX 및 반응형 동작):**
    *   **반응형 레이아웃:** 정의된 중단점(breakpoint)을 사용하여 웹사이트의 반응형 디자인을 처리하여 다양한 화면 크기에 맞게 레이아웃을 조정합니다.
    *   **페이지 로딩 애니메이션:** 초기 페이지 로딩 애니메이션 및 시각적 상태 전환을 관리합니다.
    *   **탐색 및 스크롤링:** 앵커 링크로의 부드러운 스크롤링(`jquery.scrolly.min.js`) 및 스크롤 트리거 탐색 하이라이팅(`jquery.scrollex.min.js`)을 구현하여 "스크롤스파이"와 같은 효과를 제공합니다.
    *   **모바일 탐색:** 좁은 화면에서 헤더의 사이드바 토글을 포함하여 모바일 친화적인 탐색 기능을 제공합니다.
    *   **jQuery 통합:** 일반적인 UI 개선 및 상호 작용을 위해 핵심 jQuery 라이브러리 및 다양한 jQuery 플러그인을 활용합니다.
*   **유틸리티 및 라이브러리 스크립트:**
    *   `jquery.min.js`: 애플리케이션 전반에 사용되는 기본 jQuery 라이브러리.
    *   `jquery.scrolly.min.js`, `jquery.scrollex.min.js`, `browser.min.js`, `breakpoints.min.js`, `util.js`: UI 상호 작용, 반응형 디자인 및 브라우저 호환성을 지원하는 외부 또는 사용자 지정 스크립트.
    *   `tui-grid.js`: 고급 그리드/테이블 기능을 위한 외부 JavaScript 라이브러리이지만, 제공된 `index.html` 마크업에서 현재 직접적인 사용은 최소한으로 보입니다.

### 4. `images/` (시각적 자산)
*   배너, 아바타 및 특정 계산기 아이콘(예: `calculator.png`, `mapleIcon.png`)에 사용되는 다양한 이미지 파일(`jpg`, `png`)을 포함합니다.

## 전반적인 애플리케이션 흐름:

1.  **페이지 로드:** 사용자가 웹 브라우저에서 `index.html`로 이동합니다.
2.  **리소스 로딩:** 브라우저는 HTML 구조를 로드하고, CSS 파일에서 스타일을 적용하며, JavaScript 파일을 실행합니다.
3.  **UI 초기화:** `main.js`는 일반적인 사용자 인터페이스, 반응형 동작 및 탐색 기능을 설정합니다.
4.  **계산기 초기화:** `index.js`는 모든 입력 필드의 기본값을 초기화하고, 이전에 저장된 사용자 데이터를 `localStorage`에서 로드하려고 시도하며, 기본 또는 저장된 결과를 표시하기 위한 초기 계산을 수행합니다.
5.  **사용자 상호 작용 및 실시간 업데이트:** 사용자가 입력 필드(숫자 입력, 옵션 선택)와 상호 작용하면 `index.js`는 이러한 변경 사항을 감지합니다. 그런 다음 관련 계산 함수를 실시간으로 다시 실행하고 DOM 내에서 직접 표시된 결과를 동적으로 업데이트합니다.
6.  **데이터 지속성:** 사용자 입력은 변경 시 `localStorage`에 자동으로 저장되어, 이후 애플리케이션 방문 시 사용자의 기본 설정 및 마지막으로 입력한 값이 보존됩니다.

이 시스템은 전적으로 클라이언트 측 실행, JavaScript에 의해 구동되는 동적 UI 업데이트, jQuery에 대한 강력한 의존성을 통해 DOM 조작 및 이벤트 처리를 제공하여 대화형 및 반응형 사용자 경험을 제공합니다.
